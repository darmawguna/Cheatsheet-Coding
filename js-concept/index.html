SECTION 01 Variabel & Tipe Data
📦 const — Kotak Terlakban

Dipakai untuk nilai yang tidak akan pernah berubah sepanjang program berjalan.

Analogi: Seperti nomor KTP kamu — sekali ditetapkan, tidak bisa diganti-ganti lagi!
JavaScript
const namaSiswa = "Budi"; // String (Teks) const sekolah = "SMA Negeri 1"; // namaSiswa = "Ani"; ← ERROR! tidak bisa diubah
🔄 let — Kotak Buka-Tutup

Dipakai untuk nilai yang bisa berubah selama program berjalan.

Analogi: Seperti nilai ulangan kamu — bisa naik, bisa turun setiap semester!
JavaScript
let umur = 17; // Number (Angka) let sudahMakan = true; // Boolean (true/false) let nilai = 85; umur = 18; // ✓ boleh diubah!

SECTION 02 Logika & Kondisi
🧠 if / else — Cara Website "Berpikir"

Digunakan untuk membuat keputusan berdasarkan kondisi. Seperti seorang penjaga gerbang yang mengecek syarat masuk!

Analogi: Bayangkan seperti lampu lalu lintas — JIKA lampu hijau, kamu jalan; JIKA TIDAK, kamu berhenti.
JavaScript
let nilai = 80; if (nilai >= 75) { console.log("Lulus! 🎉"); } else if (nilai >= 60) { console.log("Hampir, coba lagi! 💪"); } else { console.log("Perlu belajar lebih keras! 📚"); }

SECTION 03 Function (Fungsi)
📋 Apa Itu Fungsi?

Fungsi adalah resep — sekali ditulis, bisa dipakai berkali-kali. Tanpa fungsi, kamu harus nulis kode yang sama berulang-ulang!

Analogi: Fungsi seperti resep nasi goreng. Kamu tulis sekali, terus kapanpun mau masak, tinggal panggil resepnya — tidak perlu nulis ulang semua langkahnya!
JavaScript
// Membuat resep (Function Definition) function sapa(nama) { return "Halo " + nama + ", selamat belajar coding!"; } // Memasak (Calling Function) console.log(sapa("Andi")); // → "Halo Andi, selamat belajar coding!" console.log(sapa("Budi")); // → "Halo Budi, selamat belajar coding!"

SECTION 04 DOM Manipulation
🌐 Apa itu DOM?

DOM (Document Object Model) adalah cara JavaScript "melihat" dan "menyentuh" elemen HTML. Bayangkan HTML sebagai sebuah pohon — setiap tag adalah rantingnya, dan JS bisa mengambil serta mengubah ranting mana saja!

Analogi: HTML = buku teks. DOM = jari kamu yang bisa membuka halaman berapa saja, mencoret tulisan, atau menambah catatan di pinggir halaman secara langsung!
🎯 Menangkap Elemen
JavaScript
// Berdasarkan ID (#title) const judul = document.getElementById('title'); // Berdasarkan Class / CSS Selector const tombol = document.querySelector('.btn-submit'); // Ambil SEMUA elemen dengan class itu const semuaBtn = document.querySelectorAll('.btn');
✏️ Mengubah Elemen
JavaScript
// Ganti isi teks judul.textContent = "Halo Dunia!"; // Ubah style CSS langsung judul.style.color = "blue"; // Tambah/hapus class CSS judul.classList.add('active'); judul.classList.remove('hidden'); judul.classList.toggle('dark');

SECTION 05 Event Listener
👂 Menunggu Perintah User

Event Listener adalah cara website "mendengar" apa yang dilakukan user — klik, ketik, scroll, hover. Tanpa ini, website hanya diam dan tidak bisa bereaksi!

Analogi: Seperti pelayan restoran yang standby. Mereka tidak melakukan apa-apa sampai kamu memanggil atau memesan — itulah event listener!
JavaScript
const tombol = document.querySelector('#myButton'); // Saat tombol diklik, jalankan fungsi ini tombol.addEventListener('click', function() { alert("Tombol sudah diklik!"); document.body.style.backgroundColor = "yellow"; }); // Event lainnya yang sering dipakai: // 'mouseover' → saat cursor di atas elemen // 'keydown' → saat tombol keyboard ditekan // 'input' → saat user mengetik di input

SECTION 06 Common Debugging
🐛 Jangan Panik Saat Error!

Error adalah bagian normal dari coding. Semua developer — bahkan yang senior — masih menghadapi error setiap hari. Yang penting: tahu cara membacanya!

Analogi: Error bukan berarti kamu gagal — itu seperti dokter yang mendiagnosis penyakit. Tanpa tahu sakitnya apa, kamu tidak bisa menyembuhkannya!
  • 01
    console.log() — Senjata utama debugging. Taruh di mana saja untuk melihat isi variabel saat program berjalan. Tekan F12 → tab Console.
  • 02
    Tanda Kurung — Setiap { punya pasangan }, setiap ( punya ). Lupa menutup = error!
  • 03
    Typo & Case-Sensitive — JS membedakan huruf besar dan kecil. Nama ≠ nama ≠ NAMA. Baca error-nya baik-baik!
  • 04
    camelCase — Konvensi penamaan variabel profesional: namaLengkap, tombolKirim, nilaiAkhir. Mudah dibaca!
